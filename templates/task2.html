Network Builder task two - IP Addressing

Physical addresses are all very well for the local subnet, but if we want to communicate with computers all over the world we are going to need an IP (Internet Protocol) Address. These are used across the internet to identify computers and allow messages sent over the internet to get to where they need to go.


IP Addresses are made up of 4 numbers between 0 and 255 seperated by dots (.) - eg. '192.168.5.10'


Dynamic Host Configuration Protocol (DHCP)

How do computers get an IP address? We could set a static (fixed) address but what about if we are using a computer in a network? If everyone set their own address it would cause chaos and we could end up with all sorts of problems. To solve this, we use DHCP to send out a request to the network for an IP adress. Any servers listening will reply and assign us an IP address.
